# Cloudflare R2 — disk imajları

Büyük OS imajları ve DOS bundle'ları repoya konmaz; **Cloudflare R2**'de barındırılır
(egress ücretsiz, HTTP range request ile parça parça streaming). Küçük serbest imaj
(KolibriOS, GPL) `public/images`'te gömülü kalır — R2 olmadan da çalışır.

## 1. Bucket

1. Cloudflare panel → R2 → **Create bucket** → `cathode-images`.
2. Public access: bir **custom domain** bağla (örn. `images.cathode.example`) ya da
   R2 public bucket URL'i kullan. Range request (`Accept-Ranges: bytes`) desteklenir.
3. CORS: shell aynı origin'den değil farklı origin'den çekecekse, R2 bucket CORS'a
   sitenin origin'ini ekle (`GET`, `HEAD`, header: `Range`). Aynı origin ise gerekmez.

> COOP/COEP: shell `require-corp` çalıştığı için R2'den gelen kaynaklar
> `Cross-Origin-Resource-Policy: cross-origin` (veya CORS) döndürmeli. R2 custom
> domain'de bu header'ı bir Cloudflare Transform/Response Rule ile ekle.

## 2. İmaj hazırlama (v86)

- `.img` (raw disk) kullan. Büyük HDD imajlarını range ile çekmek için tek dosya bırak;
  isteğe bağlı olarak parçalara bölmek için `split-image.py` (copy/v86) kullanılabilir.
- Boyutu (byte) not al — `data/os.ts`'te `image.size` zorunlu (async için).

## 3. Yükleme

```sh
# rclone veya wrangler ile
wrangler r2 object put cathode-images/reactos.img --file ./reactos.img
```

## 4. Kayıt defterini güncelle

`src/data/os.ts` içinde ilgili OS:

```ts
image: { url: "https://images.cathode.example/reactos.img", async: true, size: 734003200 },
enabled: true,
```

DOS bundle'ları için aynısı `src/data/games.ts` (`bundle.url`, `enabled: true`).

## Güvenli set (spec §5)
ReactOS, FreeDOS/MS-DOS, Windows 3.x/95/98/ME/2000. **XP YOK.**
Telifli sürümler yalnızca kullanıcının kendi yüklediği imajla (BYOI) çalışır, gömülmez.
